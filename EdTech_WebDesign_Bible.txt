   CHAPTER 01

   Visual Hierarchy & Information Architecture

   Every page element competes for a prospective student's attention. Your design must instantly
   communicate what matters most -- program value, outcome, and the next action -- before any
   reading happens.

   The Priority Pyramid

   Visual hierarchy is not decoration -- it is the cognitive roadmap you draw for the user. When hierarchy breaks
   down, visitors scan frantically, find nothing compelling, and leave. On an ed-tech lead generation page, every
   element belongs to one of three tiers:

   TIER 1 Ę PRIMARY            Program headline, outcome promise, primary CTA button. One dominant element
   TIER 2 Ę SUPPORTING         per viewport. High contrast, large scale, bold weight. Nothing else competes at this
   TIER 3 Ę CONTEXTUAL         level.

                               Key benefits, program duration, key stats (salary uplift, placement rate). Medium
                               contrast, moderate size. Supports the primary without overpowering.

                               Accreditations, faculty names, fine print, breadcrumbs. Lower contrast, smallest
                               scale. Necessary but deliberately understated.

   The Seven Emphasis Tools

   Use these tools in combination. Never rely on font size alone -- that leads to headings that are too large and
   body text too small.

   Font Weight                 600Ł700 for primary, 400Ł500 for body. Never use sub-400 weights in UI text.
   Color Contrast
   Scale (Size)                Dark/rich colours for primary, lighter tones for secondary. Never grey text on
   Spatial Breathing Room      coloured backgrounds -- hand-pick a hue-matched de-saturated tone instead.
   Opacity & Saturation
   Position                    Only use drastic size jumps (18px  32px) for genuine tier-1 content. Otherwise,
   Motion (micro-interaction)  weight and colour carry the load.

                               Primary elements get more surrounding whitespace. Secondary elements are
                               packed closer together.

                               Full opacity for primary, 60Ł70% for secondary, 40Ł50% for tertiary.

                               Top-left and centre receive attention first. Tier-1 CTA belongs at the natural end of
                               the reading flow.

                               Hover states, subtle entrance animations. Primary CTAs animate; secondary
                               elements remain static.

   CHAPTER 02

   Color System & Psychology

   Color is trust, urgency, warmth, and authority -- all at once. Ed-tech must feel credible enough
   for a tuition investment yet energetic enough to inspire action. This chapter defines the full
   palette system and every rule governing its use.

   The Ed-Tech Color Philosophy

   Work exclusively in HSL (Hue, Saturation, Lightness). Hex codes are opaque -- two blues that look nearly
   identical have nothing in common in hex. HSL lets you reason about colour: 'I need this 20% lighter' or 'more
   saturated'. Define your palette up front as a fixed system -- never reach for the colour picker in the moment.

   CHAPTER 03

   Typography System

   Typography is 95% of ed-tech design. It carries the promise, the proof, and the CTA. A
   constrained, intentional type system makes your UI faster to build, easier to read, and instantly
   more credible.

   The Ed-Tech Type Scale

   Pre-define all sizes. Never nudge a font size one pixel at a time. Use px or rem -- never em (em compounds in
   nested elements, breaking your system). This scale is optimised for a university landing page at 1440px desktop
   / 390px mobile.

   Hero H1                   48Ł56px / Bold 700        Program name in hero section
   Section H2                32Ł36px / Bold 700        Section headings: Why VGU, Our Programs
   Card H3                   22Ł24px / SemiBold 600    Program card titles, feature headings
   Sub-heading H4            18Ł20px / SemiBold 600    Module names, FAQ items
   Large Body                17Ł18px / Regular 400     Hero description paragraph
   Body                      15Ł16px / Regular 400     Standard page copy -- all readable paragraphs
   Small / Caption           13Ł14px / Regular 400     Labels, metadata, dates, fine print
   Overline / Tag            11Ł12px / SemiBold 600 /  Course tags, badge labels, section pre-labels
                             UPPERCASE
   Legal / Footnote          10Ł11px / Regular 400     Disclaimer, copyright

   Font Selection Rules

   For an ed-tech brand targeting working professionals and Gen-Z learners, the personality should be: confident,
   modern, approachable, but not childish.

   Primary (Headlines)       Inter, Plus Jakarta Sans, Manrope, or DM Sans. Choose fonts with 5+ weights and
   Body / UI                 good x-height. Avoid condensed faces for body.
   Accent / Display
   Avoid                     Same family as primary at Regular 400 for consistency, or pair with a neutral sans
                             like Roboto / Open Sans.

                             A single serif (like Playfair Display) used only for testimonial pull-quotes or hero
                             sub-taglines -- adds warmth and authority.

                             Decorative fonts, fonts with <4 weights, any font that degrades below 14px. Never
                             mix more than 2 type families on one page.

   Line Length, Height & Spacing

   Readability is a conversion lever. Hard-to-read pages feel untrustworthy, and students close them. These rules
   are non-negotiable.

   Line Length               45Ł75 characters per line. Set paragraph max-width to 65Ł70ch or 600Ł680px.
   Line Height (body)        Wider content = harder to follow the line return. Tighter = feels cramped.
   Line Height (headings)
   Letter Spacing            1.6Ł1.7 for standard body copy at 15Ł16px. Wider columns need taller line-height.
   Alignment
   Baseline Alignment        1.1Ł1.2 for H1/H2. Large text needs tighter leading -- 1.5 on a 48px heading looks
                             airy and weak.

                             Tighten headlines: -0.02em to -0.03em. Normal for body. +0.06em to +0.1em for
                             ALL-CAPS tags and overlines -- caps need breathing room.

                             Left-align body copy always. Centre only for headlines 2 lines, stats numbers, or
                             standalone pull-quotes. Never centre paragraphs.

                             When mixing font sizes on one line (stat + label), align to the text baseline, not
                             vertical centre -- centred baselines create awkward optical gaps.

    CHAPTER 04

    Layout, Spacing & Grid

    Layout and spacing are the invisible architecture of your page. Generous, intentional spacing
    communicates quality, trust, and clarity -- the exact signals a prospective student needs to feel
    confident about an investment.

    The 8-Point Spacing System

    All spacing and sizing values must come from a pre-defined scale based on multiples of 4px. The 8-point base
    means consistent increments that are visually distinguishable. This eliminates decision fatigue and produces
    natural, harmonious layouts.

    4px                       Tight internal padding -- icon-to-text gaps, badge inner padding
    8px                       Minimum component padding -- small chips, tight labels
    12px                      Compact card padding, form field inner padding
    16px                      Standard component padding -- buttons (vertical), card inner
    24px                      Section sub-element gaps, list item spacing
    32px                      Card-to-card spacing, row gaps in grids
    48px                      Major section internal padding top/bottom
    64px                      Section-to-section vertical rhythm
    96px                      Hero top/bottom padding, above-the-fold breathing room
    128px+                    Full-bleed section separators on desktop

    Start With Too Much Space

    The single most reliable way to clean up any design: start by adding more space than feels necessary, then
    reduce until it feels right. White space is not empty -- it is breathing room that signals quality. Dense ed-tech
    pages feel like they are hiding something. Open pages feel like a confident institution.

    Max-Width Strategy

    Don't stretch content to fill the whole screen. Give each element the width it needs, not more.

    Page wrapper              1280px max-width, centred -- prevents ultra-wide uncomfortable line lengths
    Hero text block           640Ł720px -- keeps headline readable at all viewport widths
    Body copy paragraphs      65ch / ~600px -- the sweet spot for reading speed and comfort

    CHAPTER 05

    Depth, Shadows & Dimension

    Depth is not decoration -- it is a communication tool. Shadows tell the user what is interactive,
    what is elevated, and what is background. Used correctly, they make complex pages feel
    organised and intuitive.

    The Light Source Model

    All shadows in a single design must assume a single, consistent light source: coming from above and slightly in
    front of the screen. This means shadows are cast downward. Top edges of raised elements are lighter. Bottom
    edges are slightly darker. Violating this rule -- even subtly -- creates an uncanny, unsettling feeling.

    The Five-Level Shadow Scale

    Level                     CSS Value                                       Use Case
                                                                              Form inputs, table rows, very subtle card outlines
    Shadow 1 (flat / barely libftoexd-)shadow: 0 1px 2px rgba(15,42,74,0.08)  Standard cards, program tiles, stat boxes
                                                                              Sticky navigation on scroll, feature highlight cards
    Shadow 2 (low elevationb)ox-shadow: 0 2px 8px rgba(15,42,74,0.10)         Dropdowns, tooltips, hover-lifted cards
                                                                              Modal dialogs, brochure download popups, sticky lead forms
    Shadow 3 (medium elevbaotixo-ns)hadow: 0 4px 16px rgba(15,42,74,0.12)

    Shadow 4 (high elevatiobno) x-shadow: 0 8px 24px rgba(15,42,74,0.14)

    Shadow 5 (modal / overlbaoyx)-shadow: 0 20px 48px rgba(15,42,74,0.20)

    Depth Without Shadows -- Flat Techniques

    When your design language skews flat or neumorphic, use colour relationships to create the perception of
    depth.

    Lighter = closer                     Cards that are lighter than the page background feel raised. Use white on a slate-50
                                         background.
    Darker = recessed
    Overlap layers                       Input fields darker than the card background feel inset -- signals 'type here'.

    Solid colour shadow                  A CTA card that crosses the boundary between a dark hero section and a light body
    Border + background                  section immediately feels three-dimensional.

                                         A 2Ł3px solid offset below a card (no blur) creates a flat but tangible sense of lift.

                                         A subtle 1px SLATE-3 border on a white card on a SLATE-50 background creates
                                         crisp, legible separation without any shadow.

    Hero Sections & Above-the-Fold Design

    The hero is your 5-second window. Research shows 94% of first impressions are design-driven,
    and prospective students decide whether to stay or leave within the first scroll. Everything in the
    hero must do a specific conversion job.

    Hero Anatomy -- 7 Required Elements

    1. Pre-headline tag            Small ALL-CAPS overline (e.g. 'ONLINE MBA PROGRAM') above the headline.
                                   Sets context before the emotional hook. Use brand colour at 11px with +0.1em
                                   letter-spacing.

    2. Headline -- Outcome         The single most important line on the page. Format: [Action/Transformation] +
    Promise                        [Outcome]. Example: 'Advance to Senior Management -- In 18 Months.'
                                   48Ł56px, Bold 700, NAVY.

    3. Sub-headline                1Ł2 sentences expanding the promise with specifics. Mode of study, duration,
                                   key differentiator. 17Ł18px, Regular 400, SLATE-700.

    4. Trust proof bar             3Ł4 concise proof points below the sub-headline: '12,000+ Graduates Ę UGC
                                   Approved Ę 100% Online Ę EMI Available'. 13px SemiBold, icon-prefixed.

    5. Primary CTA                 One high-contrast Orange button: 'Apply Now' or 'Enquire Today'. Minimum
                                   48ū52px. Right of or below the sub-headline.

    6. Secondary CTA               Outlined or ghost button: 'Download Brochure'. Same line as primary. Never
                                   compete visually.

    7. Hero visual                 Right side: real student photo, video thumbnail, or illustrated outcome scene.
                                   Always apply a subtle dark overlay or colorize to ensure consistent text contrast
                                   if text is overlaid. No generic stock photos of people looking at laptops.

    CHAPTER 07

    CTA & Conversion Architecture

    Every design decision either increases or decreases the probability of a lead form submission.
    This chapter defines the conversion system: from button design to form psychology to urgency
    mechanisms that feel genuine, not manipulative.

    The CTA Button -- 8 Non-Negotiables

    1. Contrast ratio              CTA must contrast 4.5:1 against its background. Orange (#F97316) on white =
                                   3.1:1 -- use darker Orange-600 (#EA580C) for body CTAs.

    2. Size & hit target           Minimum 44ū44px (mobile), 48ū52px (desktop). Padding: 14Ł16px vertical, 24Ł32px
                                   horizontal. Never make CTAs too small to tap comfortably.

    3. Action verb first           'Apply Now', 'Enquire Today', 'Download Brochure' -- never 'Submit', 'Go', or 'Click
                                   Here'. The verb must communicate the outcome.

    4. One CTA per decision point  One primary CTA per viewport. Two primary CTAs compete and reduce clicks on
                                   both.

    5. Border radius               8Ł12px radius for approachable softness. Full-pill (9999px) for secondary ghost
                                   buttons. Maintain consistency across page.

    6. Hover state                 On hover: darken background by ~10% OR add upward shadow (2px lift). Never just
                                   underline. Feedback = confidence = clicks.

    7. Loading/processing state    Spinner or 'Submitting...' text swap. Prevents double submissions and signals
                                   responsiveness.

    8. Surrounding white space     CTA must have breathing room -- minimum 16px clear space above and below.
                                   Never crowd a CTA.

    Lead Form Optimisation

    The lead form is the most friction-filled moment in the user journey. Every unnecessary field is a leak in your
    conversion funnel. Reduce, clarify, and reassure.

    Field count                    3Ł4 fields maximum for first-touch lead capture: Name, Phone, Email, Program of
    Field labels                   Interest. Progressive profiling collects more over time.
    Input styling
    CTA button                     Always above the input, never inside as placeholder-only. Placeholder text
    Trust micro-copy               disappears on focus, leaving users confused.

                                   Inset appearance (slightly darker bg or subtle inner shadow). Active/focus state:
                                   Royal Blue border 2px. Error state: Red-600 border + icon + text.

                                   Inside the form, full-width, Orange-500. Label: 'Get Free Counselling' beats 'Submit'.
                                   Outcome-framed wins every time.

                                   Beneath the submit button: 's No spam. Your info stays private.' Removes the #1
                                   objection to form fills.

    CHAPTER 09

    Trust, Proof & Credibility Design

    Online education purchases require more trust than almost any other online transaction.
    Students are committing months of time and lakhs of rupees. Every trust signal must be visible,
    specific, and credible.

    The Trust Signal Hierarchy

    Tier 1: Institutional trust  UGC recognition, NAAC grade, AICTE approval -- above the fold or in hero trust bar

    Tier 2: Outcome trust        Placement rates, salary data, employer brand logos -- mid-page after interest is
                                 established

    Tier 3: Peer trust           Student testimonials, alumni count, cohort diversity -- emotionally reinforces the
                                 decision

    Tier 4: Process trust        Secure payment badges, privacy policy, refund policy -- near the form/pricing

    Tier 5: Expert trust         Faculty credentials, industry advisors, media mentions -- supporting, not primary

    Testimonial Design Rules

    Testimonials are not marketing copy -- they are proof. Weak testimonials ('Great course! Highly recommend!')
    hurt more than they help because they feel fabricated. Design for specificity and humanity.

    Format                       Quote  Name  Role (Before  After)  Company logo. Always include the

    Mobile-First Checklist

    Gen-Z prospective students are mobile-first. Over 70% of ed-tech traffic arrives on mobile. These are the
    non-negotiable mobile checks before any page goes live.

       s Tap targets 44ū44px -- test every button and link
       s Font size 16px inside all form inputs (prevents iOS auto-zoom)
       s Hero headline 32px on mobile -- 48px desktop headlines are too large on 390px screens
       s Single-column layout -- no horizontal scrolling anywhere
       s Fixed bottom CTA bar: 'Apply Now' always visible on program pages
       s Images: WebP format, max 200KB hero, lazy-load below-fold images
       s Page load 3 seconds on 4G -- test with Lighthouse
       s Navigation: hamburger menu with full-screen overlay. Close button visible.
       s Forms: single-column, large inputs, dropdown for program selection
       s Video: poster image while loading, controls visible, captions available
       s Spacing: 16px horizontal padding minimum on all content. Never flush-to-edge.
       s Shadow-5 modals: full-screen on mobile. Not the desktop floating-card style.
       s All hover states also have focus states (keyboard + screen reader accessibility)
       s Phone numbers: all are tap-to-call links (tel: protocol)
